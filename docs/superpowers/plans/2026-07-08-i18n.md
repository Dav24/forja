# Paso 14 — i18n Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Toda la UI y el contenido IA de Forja sale en el idioma del usuario (es/en completos), con selector funcional en Ajustes y persistencia en `profiles.language`.

**Architecture:** react-i18next con recursos estáticos y 9 namespaces por dominio en `locales/{es,en}/`; detección device→AsyncStorage→profiles.language (elección manual manda); las 3 EFs de IA leen `profiles.language` server-side e instruyen el idioma en el prompt (schema JSON intacto); push localizado con diccionario embebido en la EF.

**Tech Stack:** i18next 26 + react-i18next 17 + expo-localization 56 (ya instalados), AsyncStorage (ya instalado), Intl.DateTimeFormat (Hermes).

**Spec:** `docs/superpowers/specs/2026-07-08-i18n-design.md` (leer §6 glosario antes de traducir).

## Global Constraints

- **NativeWind v4:** estático → `className`; dinámico/colores del design system/fontFamily → `style`. Colores SIEMPRE desde `constants/colors.ts`.
- **Idiomas:** claves de idioma cortas `'es' | 'en'`. Contenido de `es` = español mexicano. `fallbackLng: 'es'`.
- **Convención de claves:** namespace = archivo (`settings.json` → `useTranslation('settings')`), claves anidadas por pantalla/bloque en camelCase: `t('account.title')`, `t('deleteAccount.confirmWord')`. En constantes se usa la forma completa `'onboarding:goals.weight_loss.title'`.
- **PROHIBIDO usar plurales de i18next** (`key_one`/`key_other` — Hermes puede no traer `Intl.PluralRules`): interpolar `{{count}}` en una sola forma redactada para funcionar con cualquier número.
- **Glosario de marca es⇄en:** OBLIGATORIO seguir la tabla §6 de la spec (Apprentice, Master Forger, FORGE TODAY, Forging since…, Vulcano y Forja no se traducen).
- **Paridad es↔en:** toda tarea que agregue claves las agrega en AMBOS idiomas y termina con `node scripts/check-i18n.mjs` + `npx tsc --noEmit` limpios.
- **Componentes de `components/ui/` no traducen por dentro** — reciben strings por props.
- **Tests de app:** no hay runner JS — verificación = tsc + check-i18n + manual. TDD real solo en EFs (Deno).
- **Supabase local corriendo** (`sg docker -c "supabase start"`); tests Deno: `cd supabase/functions && deno test .`
- **Commit al final de cada tarea. Trabajar en `master`.** Rutas relativas a `forja/`.

## Regla de extracción (aplica a Tasks 5–11)

Para cada archivo asignado: (1) identificar TODO string visible al usuario — JSX, `Alert.alert`, placeholders, labels de `Button`/`Input`, mensajes de error de handlers; (2) mover el texto español EXACTO (sin reescribirlo) a `locales/es/<ns>.json` bajo la convención de claves; (3) escribir su traducción en `locales/en/<ns>.json` siguiendo el glosario; (4) reemplazar en el código con `t('clave')` vía `const { t } = useTranslation('<ns>')`; interpolaciones con `{{var}}`. NO extraer: logs/`console.*`, claves técnicas (`'active'`, ids), emojis sueltos que acompañan a un `t()` (pueden quedar en el JSON o en JSX, criterio: si el emoji es parte de la frase va en el JSON). Textos en mayúsculas de marca (hero Bebas) se guardan ya en mayúsculas en el JSON.

```ts
import AsyncStorage from '@react-native-async-storage/async-storage';
import * as Localization from 'expo-localization';
import i18n from 'i18next';
import { initReactI18next } from 'react-i18next';
import es from '@/locales/es';
import en from '@/locales/en';

export type AppLanguage = 'es' | 'en';
export const LANGUAGE_STORAGE_KEY = 'forja.language';

export function detectDeviceLanguage(): AppLanguage {
  const code = Localization.getLocales()[0]?.languageCode ?? 'es';
  return code.startsWith('es') ? 'es' : 'en';
}

i18n.use(initReactI18next).init({
  resources: { es, en },
  lng: detectDeviceLanguage(),
  fallbackLng: 'es',
  defaultNS: 'common',
  interpolation: { escapeValue: false },
  react: { useSuspense: false },
});

// Rehidratar la elección persistida; casi siempre coincide con la detección,
// por eso no bloquea el primer render.
AsyncStorage.getItem(LANGUAGE_STORAGE_KEY)
  .then((stored) => {
    if ((stored === 'es' || stored === 'en') && stored !== i18n.language) {
      i18n.changeLanguage(stored);
    }
  })
  .catch(() => {});

export async function setAppLanguage(lang: AppLanguage): Promise<void> {
  await i18n.changeLanguage(lang);
  AsyncStorage.setItem(LANGUAGE_STORAGE_KEY, lang).catch(() => {});
}

export default i18n;
```

- [ ] **Step 1: Crear `scripts/check-i18n.mjs`:**

```js
#!/usr/bin/env node
// Verifica: (1) paridad de claves es<->en, (2) claves t() usadas que no existen,
// (3) copy en español sin migrar (heurística: acentos/¿¡ en código de UI).
import { readFileSync, readdirSync, statSync } from 'node:fs';
import { join, relative } from 'node:path';
import { fileURLToPath } from 'node:url';

// fileURLToPath y no .pathname: la ruta del repo tiene espacios ("Physis Labs")
const ROOT = fileURLToPath(new URL('..', import.meta.url));
const NAMESPACES = ['common', 'auth', 'onboarding', 'home', 'chat', 'plans', 'progress', 'profile', 'settings'];
const SRC_DIRS = ['app', 'components', 'hooks', 'constants', 'lib'];
const allowlist = JSON.parse(readFileSync(join(ROOT, 'scripts/check-i18n-allowlist.json'), 'utf8'));

function flatten(obj, prefix = '') {
  return Object.entries(obj).flatMap(([k, v]) => {
    const key = prefix ? `${prefix}.${k}` : k;
    return typeof v === 'object' && v !== null ? flatten(v, key) : [key];
  });
}

function loadNs(lang) {
  const out = {};
  for (const ns of NAMESPACES) {
    out[ns] = new Set(flatten(JSON.parse(readFileSync(join(ROOT, `locales/${lang}/${ns}.json`), 'utf8'))));
  }
  return out;
}

function* walk(dir) {
  for (const name of readdirSync(dir)) {
    const p = join(dir, name);
    if (statSync(p).isDirectory()) yield* walk(p);
    else if (/\.(tsx?|ts)$/.test(name) && !name.endsWith('.d.ts')) yield p;
  }
}

const es = loadNs('es');
const en = loadNs('en');
const errors = [];

// (1) Paridad
for (const ns of NAMESPACES) {
  for (const k of es[ns]) if (!en[ns].has(k)) errors.push(`[paridad] en/${ns}.json falta: ${k}`);
  for (const k of en[ns]) if (!es[ns].has(k)) errors.push(`[paridad] es/${ns}.json falta: ${k}`);
}

// (2) Claves usadas vs definidas
const KEY_RE = /\bt\(\s*['"]([A-Za-z0-9_.:-]+)['"]/g;
const LABELKEY_RE = /\b(?:labelKey|titleKey|descriptionKey)\s*:\s*['"]([A-Za-z0-9_.:-]+)['"]/g;
const files = SRC_DIRS.flatMap((d) => [...walk(join(ROOT, d))]);
for (const file of files) {
  const src = readFileSync(file, 'utf8');
  const rel = relative(ROOT, file);
  for (const re of [KEY_RE, LABELKEY_RE]) {
    re.lastIndex = 0;
    let m;
    while ((m = re.exec(src))) {
      const raw = m[1];
      if (raw.includes(':')) {
        const [ns, key] = [raw.slice(0, raw.indexOf(':')), raw.slice(raw.indexOf(':') + 1)];
        if (!NAMESPACES.includes(ns)) { errors.push(`[clave] ${rel}: namespace desconocido '${ns}' en '${raw}'`); continue; }
        if (!es[ns].has(key)) errors.push(`[clave] ${rel}: '${raw}' no existe en es/${ns}.json`);
      } else if (!NAMESPACES.some((ns) => es[ns].has(raw))) {
        errors.push(`[clave] ${rel}: '${raw}' no existe en ningún namespace`);
      }
    }
  }
}

// (3) Copy español sin migrar (solo UI: app/ y components/)
const SPANISH_RE = /['"`>][^'"`<\n]*[áéíóúñÁÉÍÓÚÑ¿¡][^'"`<\n]*/;
for (const file of files) {
  const rel = relative(ROOT, file);
  if (!/^(app|components)\//.test(rel)) continue;
  if (allowlist.files.includes(rel)) continue;
  const lines = readFileSync(file, 'utf8').split('\n');
  lines.forEach((line, i) => {
    const t = line.trim();
    if (t.startsWith('//') || t.startsWith('*') || t.startsWith('/*')) return;
    if (allowlist.patterns.some((p) => line.includes(p))) return;
    if (SPANISH_RE.test(line)) errors.push(`[copy] ${rel}:${i + 1}: posible español sin migrar: ${t.slice(0, 80)}`);
  });
}

if (errors.length) {
  console.error(`check-i18n: ${errors.length} problema(s):\n` + errors.join('\n'));
  process.exit(1);
}
console.log('check-i18n: OK');
```

- [ ] **Step 1: Crear `hooks/useSyncLanguage.ts`:**

```ts
import { useEffect, useRef } from 'react';
import { useAuthStore } from '@/store/auth.store';
import { useProfile, useUpdateProfile } from '@/hooks/useProfile';
import i18n, { setAppLanguage, type AppLanguage } from '@/lib/i18n';

// profiles.language manda cuando existe; si es NULL (alta nueva o usuario
// pre-i18n) se escribe UNA vez el idioma activo de la app.
export function useSyncLanguage() {
  const { user } = useAuthStore();
  const { data: profile } = useProfile();
  const updateProfile = useUpdateProfile();
  const wroteInitial = useRef(false);

  useEffect(() => {
    if (!user || profile === undefined || profile === null) return;
    const lang = profile.language;
    if (lang === 'es' || lang === 'en') {
      if (lang !== i18n.language) setAppLanguage(lang as AppLanguage);
    } else if (!wroteInitial.current) {
      wroteInitial.current = true;
      updateProfile.mutate({ language: i18n.language as AppLanguage });
    }
  }, [user, profile, updateProfile]);
}
```

## Notas de riesgo conocidas

- **`react: { useSuspense: false }` es obligatorio** — sin él, useTranslation suspende y RN no tiene Suspense boundary configurado.
- **Hermes e `Intl.PluralRules`:** por eso la Global Constraint de no usar plurales i18next.
- **Cache del prompt de chat:** `languageLine` va en el bloque de contexto de usuario (ya es per-user), NO en `SYSTEM_PROMPT` (compartido y cacheado).
- **Task 9 es la más grande** — el implementador puede reportar DONE_WITH_CONCERNS para partirla; mejor eso que una pasada descuidada.
- **check-i18n `[copy]` es heurística:** frases españolas sin acentos (p. ej. "Continuar") no las detecta — la migración depende de la revisión por tarea, el script es red de seguridad, no garantía.
- **`profiles.language` ya existe en DB y `useUpdateProfile` ya lo acepta** — no hay migración SQL en este paso.
